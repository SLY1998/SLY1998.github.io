    # 基础概念

    # JVM vs JDK vs JRE

    • Java Virtual Machine（java虚拟机），运行 Java 字节码的虚拟机
    • Java Development Kit（java开发工具包），提供给开发人员使用的工具，用于创建和编译 Java 程序。包含JRE、javac、javadoc、jdb（调试器）、jconsole、javap（反编译工具）等
    • Java Runtime Environment（java运行环境），运行 Java 程序的环境，主要包含两个部分：JVM、Java基础类库（Class Library）

    从 JDK 9 开始，就不需要区分 JDK 和 JRE 的关系了，取而代之的是模块系统。在引入了模块系统之后，JDK 被重新组织成 94 个模块。Java 应用可以通过新增的 jlink 工具，创建出只包含所依赖的 JDK 模块的自定义运行时镜像。这样可以极大的减少 Java 运行时环境的大小。

    # AOT vs JIT（记不住）

    • AOT（Ahead Of Time Compilation），静态编译，在程序运行前，将代码编译成机器码，运行时直接加载机器码，无需解释器。GraalVM
    • JIT（Just In Time Compilation），运行时编译

    # 基本语法

    # 移位运算符（看不懂）

    • << 左移运算符，将一个数的二进制位向左移动指定的位数，右侧补 0。
    • >> 右移运算符，将一个数的二进制位向右移动指定的位数，左侧补 0。
    • >>> 无符号右移，忽略符号位，空位都以 0 补齐

    # 基本数据类型

    byte、short、int、long能表示的最大正数都减 1 了，这是为什么呢？

    在二进制补码表示法中，最高位是用来表示符号的（0 表示正数，1 表示负数），其余位表示数值部分。所以，如果我们要表示最大的正数，我们需要把除了最高位之外的所有位都设为 1。如果我们再加 1，就会导致溢出，变成一个负数。

    所有整型包装类对象之间值的比较，全部使用 equals 方法比较

    # 什么是自动拆装箱

    • 装箱：将基本类型用它们对应的引用类型包装起来；
    • 拆箱：将包装类型转换为基本数据类型；
    Integer i = 10;  //装箱
int n = i;   //拆箱

    2

    如果频繁拆装箱的话，也会严重影响系统的性能。我们应该尽量避免不必要的拆装箱操作。

float b = 1.8f - 1.7f;
System.out.printf("%.9f",a);// 0.100000024
System.out.println(b);// 0.099999905
System.out.println(a == b);// false
