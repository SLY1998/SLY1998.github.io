    # 什么是Web Worker

    Web Worker是HTML5提供的一种JavaScript运行机制，允许在后台线程中运行JavaScript代码，而不会阻塞主线程。这使得我们可以执行耗时的计算任务，而不会影响页面的响应性。

    # Web Worker的类型

    # 1. Dedicated Worker（专用Worker）

    • 只能被创建它的脚本访问
    • 使用new Worker()创建

    # 2. Shared Worker（共享Worker）

    • 可以被多个脚本访问，甚至是不同的窗口、iframe或其他Worker
    • 使用new SharedWorker()创建

    # 3. Service Worker（服务Worker）

    • 主要用于离线缓存、推送通知和后台同步
    • 运行在浏览器和网络之间的代理

    # 数据传递

    # 1. 结构化克隆

    • Web Worker使用结构化克隆算法传递数据
    • 支持大多数JavaScript数据类型
    • 不支持函数、DOM节点等

    # 2. 转移所有权（Transferable Objects）

    • 对于大型数据，可以使用转移所有权的方式提高性能
    • 转移后，原线程不再拥有该数据的访问权
    // 主线程
const buffer = new ArrayBuffer(1024);
worker.postMessage(buffer, [buffer]);

    # Web Worker的限制

    1. 同源策略：Worker脚本必须与主线程脚本同源
    2. 无法访问DOM：Worker不能直接操作DOM
    3. 无法使用某些全局对象：如window、document等
    4. 通信限制：只能通过消息传递进行通信
    5. 文件限制：不能直接读取本地文件系统

    # 浏览器支持

    • Chrome 4+
    • Firefox 3.5+
    • Safari 4+
    • Edge 12+
    • IE 10+

    # Shared Worker内部

    // shared-worker.js
const connections = [];

self.addEventListener('connect', (event) => {
  const port = event.ports[0];
  connections.push(port);
  
  port.addEventListener('message', (event) => {
    console.log('Shared worker received:', event.data);
    
    // 向所有连接的端口广播消息
    connections.forEach((conn) => {
      conn.postMessage(`Broadcast: ${event.data}`);
    });
  });
  
  port.start();
});

    # Service Worker简介

    Service Worker是一种特殊类型的Worker，主要用于：

    1. 离线缓存：缓存资源，实现离线访问
    2. 推送通知：接收服务器推送的通知
    3. 后台同步：在后台同步数据
    4. 网络请求拦截：拦截和处理网络请求

    # 注册Service Worker

    if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/service-worker.js')
      .then((registration) => {
        console.log('Service Worker registered with scope:', registration.scope);
      })
      .catch((error) => {
        console.error('Service Worker registration failed:', error);
      });
  });
}

    # 最佳实践

    1. 只在必要时使用Worker：创建Worker有一定的性能开销
    2. 合理设计通信机制：避免频繁的消息传递
    3. 处理Worker的错误：始终添加错误监听器
    4. 及时关闭不需要的Worker：释放资源
    5. 使用Transferable Objects处理大数据：提高性能
    6. 考虑浏览器兼容性：为不支持的浏览器提供降级方案

    # 总结

    Web Worker为JavaScript提供了多线程能力，使得我们可以在后台执行耗时任务，提高页面的响应性。通过合理使用Web Worker，可以显著提升Web应用的性能和用户体验。
