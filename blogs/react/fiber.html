    【react】浅谈fiber架构

    凡年换酒柴1998🌸 2022/5/10 react

    React Fiber 是 React 16 引入的核心算法重构，旨在解决大型应用渲染性能问题并支持异步渲染等新特性。其核心思想是通过可中断、优先级调度和增量渲染，优化用户交互体验。以下是 Fiber 架构的关键点：

    # 1. 旧架构的瓶颈

    • Stack Reconciler：递归遍历虚拟DOM树，无法中断任务，长时间任务会导致主线程阻塞（如动画卡顿）。
    • 同步渲染：一旦开始渲染，必须完成整个树的计算，无法响应用户交互（如输入延迟）。

    # 2. Fiber 的核心设计

    # （1）Fiber 节点

    • 将虚拟DOM节点抽象为Fiber节点，保存组件类型、状态、副作用等信息。
    • 数据结构：链表结构（child, sibling, return），替代树形结构，实现可中断遍历。

    # （2）任务分割与调度

    • 增量渲染：将渲染任务拆分为多个小单元（Fiber节点处理），通过浏览器空闲时间（requestIdleCallback）分片执行。
    • 优先级调度：为不同更新（用户交互 > 动画 > 数据加载）分配优先级，高优先级任务可打断低优先级任务。

    # （3）双缓存机制

    • Current Tree：当前显示的UI状态。
    • WorkInProgress Tree：后台构建的新状态，完成后直接切换，减少渲染抖动。

    # 3. Fiber 的关键能力

    • 可中断与恢复：渲染过程可暂停，优先处理用户交互，再恢复剩余任务。
    • 并发模式（Concurrent Mode）：支持异步渲染，实现更流畅的交互（如Suspense延迟加载）。
    • 错误边界：通过组件生命周期（如static getDerivedStateFromError）捕获子树错误。

    # 4. 生命周期的影响

    • 废弃生命周期：componentWillMount, componentWillUpdate等可能被多次调用（因任务中断），由getDerivedStateFromProps等替代。
    • 副作用分离：useEffect替代componentDidMount等，支持更灵活的副作用管理。

    # 5. 对开发者的意义

    • 更流畅的UI：动画、输入等高优先级操作响应更快。
    • 复杂应用优化：大数据量列表、懒加载等场景性能提升。
    • 未来特性基础：支持并发模式（Concurrent Mode）、服务器组件（Server Components）等。

    # 总结

    React Fiber 通过重新设计协调算法，将同步渲染改为异步可中断调度，解决了卡顿问题，并为 React 的长期演进奠定了基础。开发者无需直接操作 Fiber，但理解其原理有助于优化应用性能。
