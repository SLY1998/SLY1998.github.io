    # 第一阶段：夯实基础（3-5 天）

    # 1. HTML/CSS 核心

    • 语义化标签与 SEO 优化原则

      • 具有明确意义的 HTML 标签，如 header、nav、article、section 等。提高可读性、可访问性，便于搜索引擎抓取；助力 SEO 优化；提高网站用户体验。
      • 内容优化，技术优化，外部优化，用户体验。
      • 语义化标签是 SEO 优化的技术基础，通过清晰的代码结构和内容描述帮助搜索引擎理解页面。但 SEO 是一个系统工程，需结合内容质量、技术优化和用户体验等多维度策略。

    • 盒模型（BFC/IFC/FFC/GFC）

      • 盒模型是 CSS 布局的基础，每个元素都可以看作是一个矩形的盒子，包含内容、内边距、边框和外边距。
      • 标准盒模型（content-box）：元素的宽度和高度仅包括内容区域，默认情况下使用此模型。
      • IE 盒模型（border-box）：元素的宽度和高度包括内容区域、内边距和边框。
      • BFC（Block Formatting Context）：块级格式化上下文，内部的块级元素会在垂直方向上一个接一个地放置，并且与外部元素隔离。
      • IFC（Inline Formatting Context）：内联格式化上下文，内部的行内元素会在水平方向上一个接一个地放置，并且与外部元素隔离。
      • FFC（Flex Formatting Context）：弹性格式化上下文，内部的元素会根据 flex 属性进行布局，并且与外部元素隔离。
      • GFC（Grid Formatting Context）：网格格式化上下文，内部的元素会根据 grid 属性进行布局，并且与外部元素隔离。

    • 布局方案：Flex/Grid/响应式设计（媒体查询、移动端 1px 问题）

    • CSS 层叠与继承机制

      1. 层叠机制
      • 重要性（Importance）：由!important 标记的样式规则优先级最高。
      • 特异性（Specificity）：选择器的特异性决定了规则的优先级。内联样式（Inline styles） > ID 选择器 > 类选择器、属性选择器和伪类 > 元素选择器和伪元素
      • 源顺序（Source Order）：当重要性相同且特异性相同时，样式表中后出现的规则将覆盖先出现的规则。
      1. 继承机制
      • 可继承属性,如 color、font-family 等，子元素会自动从父元素继承这些属性的值。
      • 不可继承属性,如 border、margin 等，子元素不会从父元素继承这些属性的值，需要显式定义。
      • 继承控制,使用 inherit 关键字可以使某个属性明确地从父元素继承，即使它通常是不可继承的。使用 initial 关键字可以将属性设置为它的初始值，忽略任何继承或层叠的影响。

    • 性能优化：重绘与回流、CSS 渲染阻塞解决方案

      1. 重绘与回流
      • 重绘（Repaint）：当元素的外观发生变化但几何位置不变时，会触发重绘。
      • 回流（Reflow）：当元素的几何位置发生变化时，会触发回流。
      • 重绘与回流的性能影响：频繁的重绘和回流会导致页面性能下降，应尽量避免。
      1. CSS 渲染阻塞解决方案

        当浏览器解析 HTML 文档时，遇到 <link> 标签加载外部 CSS 文件会暂停 DOM 构建和渲染，直到 CSSOM（CSS Object Model）构建完成。这会导致页面“白屏”，影响用户体验。

      • 减少关键路径资源：内联关键 CSS，延迟非关键 CSS
      • 异步加载 CSS，使用 JavaScript 动态插入样式表，避免阻塞渲染
      • 优化 CSS 文件，压缩和合并 CSS 文件，使用 CDN
      • 使用媒体查询，通过<link>标签的 media 属性，为不同的媒体类型（如屏幕、打印等）加载不同的 CSS 文件。
      • 预加载关键资源，使用 <link rel="preload"> 提前加载关键资源，提高加载效率

    1. 执行上下文：一段代码执行所需的所有信息

  },

 VariableEnvironment = { ... },

    # 第二阶段：核心知识体系（5-7 天）

    1. 浏览器工作原理

    • 关键渲染路径（Critical Rendering Path）

    • 浏览器缓存策略（强缓存/协商缓存）

    • 安全机制（CSP/XSS/CSRF 防护）

    • 浏览器存储方案对比（Cookie/WebStorage/IndexedDB）

    • 渲染引擎工作流：合成层（Composite Layer）优化策略

    • 内存管理实战：Heap Snapshot 分析内存泄漏

    • WebAssembly 初探：与 JS 的性能对比测试

    1. 网络协议

    • HTTP/1.1 vs HTTP/2 vs HTTP/3 核心差异

    • HTTPS 握手过程与加密原理

    • WebSocket 协议实现机制

    • WebSocket 优化：重连策略与心跳机制

    • CDN 原理与性能优化实践

    1. 性能优化体系

    • Lighthouse 性能指标解读（LCP/FID/CLS/TBT）

    • 代码分割与 Tree Shaking 实现原理

    • 图片优化方案（WebP/懒加载/渐进加载）

    • Service Worker 与离线化策略

    # 第三阶段：框架与工程化（5-7 天）

    1. 主流框架深度

    • React：Fiber 架构原理、Hooks 实现机制、diff 算法优化

    • Vue：响应式原理（Proxy vs defineProperty）、编译时优化

    • 框架对比：React/Vue 设计哲学差异、SSR 实现方案、虚拟 DOM 差异算法白板推演

    1. 状态管理

    • Redux 中间件机制（redux-thunk/redux-saga）

    • Vuex 与 Pinia 架构对比

    • 状态管理选型策略（何时需要状态管理？）

    1. 工程化体系

    • Webpack 构建流程（Loader/Plugin 开发）

    • Babel 编译原理（AST 操作）

    • 模块化规范（ES Module 与 CommonJS 差异）

    • CI/CD 流程设计与监控体系

    1. 现代前端架构

    • 微前端沙箱机制：qiankun 源码分析

    • 状态管理演进：RxJS 与 XState 状态机实践

    • 低代码平台设计：DSL 与可视化编排原理

    # 第四阶段：实战与算法（3-5 天）

    1. 项目深度复盘

    • 准备 2-3 个复杂项目案例（技术选型论证、性能优化指标）

    • 架构设计能力体现（微前端/qiankun 方案、Monorepo 实践）

    • 异常监控体系（Sentry 原理、SourceMap 解析）

    1. 算法与数据结构

    • 高频算法题：链表操作、二叉树遍历、动态规划

    • 前端特定算法：虚拟 DOM diff、LRU 缓存实现

    • 使用 LeetCode/CodeTop 进行针对性训练（建议 Easy-Medium 每日 5 题）

    # 第五阶段：模拟面试与软技能（2-3 天）

    1. 行为面试准备

    • 技术决策案例（如技术方案对比、技术债务处理）

    • 团队协作经验（Code Review 规范、跨部门协作）

    • 学习方法论（如何跟进新技术？）

    1. 模拟面试

    • 使用 Pramp/牛客网进行模拟面试

    • 录制技术分享视频（考察表达逻辑）

    • 准备技术反问清单（体现岗位认知深度）

    # 专项加分项（持续关注）

    1. 前沿技术

    • WebAssembly 应用场景

    • Serverless 在前端的落地

    • 低代码平台架构设计

    1. TypeScript 进阶

    • 类型体操训练（Utility Types）

    • 装饰器元编程实践

    • TS 编译器原理

    1. AI 增强开发

    • AI 代码生成：Copilot 精准 Prompt 编写技巧

    • 智能 UI 生成：通过 GPT-4 生成 DSL 转代码

    • 自动化测试：AI 视觉回归测试（Playwright）

    • 书籍：《JavaScript 高级程序设计》《深入 React 技术栈》《Web 性能权威指南》

    # 每日执行建议

    • 上午：专题知识深度学习（结合手写代码验证）

    • 下午：框架源码阅读+算法训练

    • 晚上：项目复盘+模拟面试

    • 持续：参与 GitHub 开源项目/技术博客输出（建立个人技术品牌）
